介绍一下hexo的文件架构

  • public 最终所见网页的所有内容
  • node_modules 插件以及hexo所需node.js模块
  • scaffolds 模板文件夹，新建文章，会默认包含对应模板内容
  • themes 存放主题文件
  • source 用于存放你的文章
  • _config.yml 站点配置文件，设定一些公开信息等

部署到GitHub

当然你得有一个 GitHub 账号，具体我就不详细说了，直接开始。

创建GitHub个人仓库

新建仓库，仓库名一定为：用户名.github.io，保证仓库public。

GitHub Pages配置

GitHub Pages 是一种由 GitHub 中的仓库/项目直接创建的网页。管理简单，在本地编辑仓库中的内容，上传到 GitHub 上，GitHub Pages 就能快速完成更新，重点是不花钱，不花钱，不花钱。

在仓库设置中找到Pages

生成SSH公钥，添加到Github

右键桌面打开 git bash here

创建SSH密钥，一路回车。

它会告诉你密钥保存的路径，找到刚刚生成的 id_rsa.pub，用记事本打开，复制文件全部内容。

打开github，右边点击头像 -> 设置 -> SSH and GPG keys - > New SSH key
Title随意，粘贴密钥到key
